# Roadmap по улучшению продукта — kztusdt.kz

Документ-план развития OTC-брокера **KZT ↔ USDT** (Laravel 13 + Vue 3/Inertia, кастодиальные HD-кошельки BEP20/TRC20, двойная бухгалтерия, ручное подтверждение фиата).

Приоритеты:

- **P0** — критично: безопасность средств и денежная целостность. Делать в первую очередь.
- **P1** — операционная зрелость: то, без чего нельзя надёжно эксплуатировать финпродукт.
- **P2** — продукт и масштабирование: снятие потолка по объёму, развитие.
- **P3** — инженерные улучшения: качество кода, поддерживаемость.

Оценка трудозатрат: S = 1–3 дня, M = до 1–2 недель, L = месяц+.

---

## P0 — Критично (безопасность и деньги)

### P0-1. Управление ключами кошелька — `L`
Сейчас корень доверия — `WALLET_MNEMONIC` в `.env` (plaintext). Компрометация сервера = полная потеря средств.

- [ ] Вынести мнемонику из `.env` в KMS/секрет-менеджер (HashiCorp Vault / AWS KMS / GCP KMS).
- [ ] Разделить hot / warm / cold storage: на сервере держать только hot-wallet с лимитом; крупные суммы — в cold.
- [ ] Дорожная цель: MPC / threshold-подпись (Fireblocks или self-hosted, напр. `tss-lib`) либо шардирование ключа.
- [ ] Лимиты на автоматические выводы; суммы выше порога — только с ручным мультиподтверждением.

**Критерий готовности:** приватный ключ недоступен в plaintext на проде; подпись транзакций изолирована.

### P0-2. Дефолтные учётки и 2FA для staff — `S`
Сидер создаёт `admin@exchange.local` / `security@exchange.local` с известными паролями.

- [ ] Принудительная смена пароля при первом входе.
- [ ] Блокировка запуска сидера в `production` (или генерация случайного пароля с выводом один раз).
- [ ] TOTP-2FA (Google Authenticator) обязательна для всех admin/security ролей.

### P0-3. Аудит вебхуков и доверенных прокси — `S`
- [ ] Проверить, что у **всех** `/api/*` с отключённым CSRF (Sumsub, Aitu, push, legacy telegram) есть валидация подписи/HMAC.
- [ ] Удалить CSRF-исключение для несуществующего `api/auth/telegram/webhook`.
- [ ] Сузить `trustProxies(at: '*')` до подсети nginx/балансировщика.

## P1 — Операционная зрелость

### P1-1. CI/CD — `M`
Сейчас нет ни Docker, ни пайплайнов; тесты, линт и сборка — вручную.

- [ ] GitHub Actions: `pint` (lint) + `phpunit` + `npm run build` на каждый PR.
- [ ] `Dockerfile` + `docker-compose` для воспроизводимого окружения (php-fpm, postgres, redis).
- [ ] Скрипт zero-downtime деплоя: миграции, `queue:restart`, `config:cache`, прогрев.

### P1-2. Observability — `M`
Нет Sentry / Telescope / Horizon.

- [ ] Sentry — трекинг ошибок и алертов.
- [ ] Horizon — мониторинг очередей (`withdrawals:process`, `sweep:run`, `CreateWalletAfterKycApproved`).
- [ ] Алерты на: застрявшие выводы в `needs_reconcile`, отставание индексатора депозитов, протухшие курсы (`rates:refresh`), расхождение баланса.

### P1-3. Демон сверки (reconciliation) — `M`
Для OTC с ручными операциями — обязательная защита от расхождений.

- [ ] Ежедневная сверка: сумма on-chain балансов кошельков ↔ `ledger_entries`.
- [ ] Отчёт о расхождениях + алерт при превышении порога.

### P1-4. Очереди в проде — `S`
- [ ] `queue:listen` сейчас только в `composer dev`. Настроить supervisor / Horizon worker в деплое.

## P2 — Продукт и масштабирование

### P2-1. Автоматизация фиата — `L`
Узкое место: каждый KZT-платёж подтверждается человеком — потолок по объёму.

- [ ] Интеграция с банковским / платёжным API (Kaspi, Halyk) для автосверки поступлений по назначению платежа.
- [ ] Автоподтверждение по сумме + референсу; человек — только спорные кейсы.

### P2-2. Чистка легаси и синхронизация документации — `S`
- [ ] README описывает Telegram OTP, реально работает WhatsApp OTP — привести в соответствие.
- [ ] Удалить `TelegramWait.vue`, таблицу `user_telegram_accounts`, Breeze-остатки (Register/VerifyEmail).
- [ ] Решить судьбу неиспользуемого Sanctum: убрать или задействовать для API.

### P2-3. Multi-tenant / white-label — `M`
Есть `tenants` + `tenant_id` + `exchange_point_id` на ролях, но без таблицы `exchange_points`.

- [ ] Если white-label в планах — достроить (таблица точек, скоупинг, изоляция данных).
- [ ] Если нет — убрать scaffolding, чтобы не вводить в заблуждение.

## P3 — Инженерные улучшения

### P3-1. Авторизация через Policies — `M`
- [ ] Перейти с middleware-only на Laravel **Policies** для per-resource правил (выводы, ордера, кошельки) — легче аудировать.

### P3-2. Выделенный API-слой — `M`
- [ ] Вынести вебхуки и OTP из `web.php` в `routes/api.php` с версионированием (`/api/v1/`) и API Resources — задел под мобильное приложение.

### P3-3. Тесты и покрытие — `S`
- [ ] Включить coverage-репорт в `phpunit.xml` (сейчас 214 тестов, метрики покрытия нет).
- [ ] Добавить фабрики вместо хелперов в `tests/Concerns`.
- [ ] Убрать упоминания удалённых Telegram-тестов из `.phpunit.result.cache`.
