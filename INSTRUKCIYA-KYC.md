# Инструкция: внешний KYC через Sumsub — простым языком

## Что это такое

Сейчас проверка документов (KYC) у вас **ручная**: клиент грузит фото, сотрудник СБ смотрит глазами и жмёт «Одобрить»/«Отклонить». Это бесплатно и для старта достаточно.

**Sumsub** (https://sumsub.com) — это внешний сервис, который проверяет документы **автоматически**: распознаёт паспорт, сверяет селфи с фото в документе, проверяет по санкционным спискам. Решение приходит обычно за 1–2 минуты, без участия вашего сотрудника.

Поддержка Sumsub в ваш сайт **уже встроена**. Чтобы её включить, нужен только аккаунт Sumsub и 4 ключа в настройках.

## Сколько стоит

- Это **платный** сервис: примерно **$1–2 за одну проверку** человека (зависит от тарифа и страны).
- Есть **бесплатный пробный период (trial)** — хватит, чтобы протестировать.
- Пока ключи не вписаны — ничего не ломается: работает обычный ручной KYC.

## Шаг 1. Регистрация

1. Откройте https://sumsub.com и нажмите «Get started» / «Try for free».
2. Зарегистрируйтесь как компания (понадобится email).
3. После входа вы попадёте в **дашборд** (панель управления) Sumsub.

## Шаг 2. Уровень проверки (level)

1. В дашборде: **Integrations → Applicant levels** (или просто «Levels»).
2. Там обычно уже есть готовый уровень `basic-kyc-level` (паспорт + селфи). Если нет — создайте уровень и запомните его **точное имя**.
3. Если имя отличается от `basic-kyc-level` — его нужно будет вписать в настройки (см. шаг 4).

## Шаг 3. Ключи API

1. В дашборде: **Dev space → App tokens** (раздел для разработчиков).
2. Нажмите «Create app token» (тип Server).
3. Вам выдадут **два значения**:
   - **App token** — строка вида `sbx:abcd1234...` (sandbox) или `prd:...` (боевой);
   - **Secret key** — длинная секретная строка (показывается один раз — сохраните!).

## Шаг 4. Вписать ключи на сервере

Откройте файл `/var/www/crypto-exchange/backend/.env` и найдите строки (они уже там есть, пустые):

```
KYC_PROVIDER=manual
SUMSUB_APP_TOKEN=
SUMSUB_SECRET_KEY=
SUMSUB_LEVEL_NAME=basic-kyc-level
SUMSUB_WEBHOOK_SECRET=
```

Впишите:

```
KYC_PROVIDER=sumsub                 ← включает Sumsub вместо ручного KYC
SUMSUB_APP_TOKEN=ваш_app_token
SUMSUB_SECRET_KEY=ваш_secret_key
SUMSUB_LEVEL_NAME=basic-kyc-level   ← имя уровня из шага 2
SUMSUB_WEBHOOK_SECRET=придумайте_длинный_пароль_см_шаг_5
```

Потом выполните на сервере:

```bash
cd /var/www/crypto-exchange/backend
php artisan config:cache
systemctl restart php8.3-fpm crypto-queue
```

## Шаг 5. Webhook (чтобы сайт узнавал результат проверки)

Webhook — это «звонок» от Sumsub вашему сайту: «проверка такого-то клиента завершена, результат — одобрено/отклонено».

1. В дашборде Sumsub: **Dev space → Webhooks** → «Create webhook».
2. **URL**: `https://kztusdt.kz/api/kyc/sumsub/webhook`
3. **Тип событий**: отметьте `applicantReviewed` (результат проверки).
4. **Secret key для подписи** (digest secret): придумайте длинную случайную строку (например 40+ символов), укажите её там **и эту же строку** впишите в `.env` в `SUMSUB_WEBHOOK_SECRET=`. По этой строке сайт проверяет, что запрос действительно от Sumsub.
5. Снова `php artisan config:cache` после правки `.env`.

## Как это будет работать для клиента

1. Клиент заходит на https://kztusdt.kz/kyc.
2. Вместо ручной анкеты открывается окно Sumsub: сфотографировать документ, сделать селфи.
3. Sumsub проверяет автоматически и присылает результат на сайт.
4. При одобрении: статус KYC становится «approved», **кошелёк создаётся автоматически** — так же, как при ручном одобрении. Клиенту приходит уведомление в Telegram.
5. При отказе: клиент видит причину и может пройти проверку заново.

## Важно знать

- **Пока ключи не вписаны** (или `KYC_PROVIDER=manual`) — работает прежний ручной KYC через `/admin/kyc`. Ничего настраивать не обязательно прямо сейчас.
- Sumsub выдаёт сначала **sandbox** (тестовые) ключи — с ними проверки не настоящие, удобно тестировать. Боевые ключи дадут после заключения договора/оплаты.
- Ручная админка `/admin/kyc` никуда не девается — её можно использовать параллельно (например, для спорных случаев).
- Если что-то не работает: смотрите логи `storage/logs/laravel.log` — ошибки Sumsub пишутся туда со словом «Sumsub».
